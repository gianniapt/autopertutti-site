# Visual Hierarchy & Layout

Questa pagina definisce la gerarchia visiva consigliata per la homepage di Autopertutti: cosa mostrare per primo, come distribuire i blocchi, dove inserire le CTA e come mantenere equilibrio tra vendita, noleggio e servizi.

# Principio UX principale

La homepage deve funzionare come un router intelligente: non deve spingere solo una categoria, ma aiutare l’utente a capire subito dove andare.

Obiettivo entro i primi 5 secondi:

- capire cosa offre Autopertutti
- scegliere tra vendita, noleggio o servizi
- trovare un modo rapido per contattare il team

# Above the fold

La prima schermata è la più importante. Deve contenere:

- messaggio chiaro
- tre percorsi principali
- CTA visibili
- immagine forte
- contatto rapido

## Layout desktop consigliato

```
--------------------------------------------------
Header: Logo | Auto | Moto | Noleggio | Servizi | Contatti | WhatsApp
--------------------------------------------------
Hero left: headline + testo + 3 CTA
Hero right: immagine / collage / veicolo + officina
--------------------------------------------------
Quick selector: Acquisto | Noleggio | Servizi | Consulente
--------------------------------------------------
```

## Layout mobile consigliato

```
Logo + menu + WhatsApp
Headline
Subheadline
CTA 1
CTA 2
CTA 3
Immagine
Quick selector verticale
```

# Gerarchia visiva dei blocchi

## Priorità 1 — Chiarezza immediata

Headline consigliata:

```
Auto, moto, noleggio e servizi: tutto in un unico posto.
```

Questa frase deve essere grande, leggibile e immediata.

## Priorità 2 — Scelta rapida

Subito sotto o subito dopo la hero:

```
Di cosa hai bisogno oggi?
```

Quattro card:

1. Acquistare un veicolo
2. Noleggiare un veicolo
3. Prenotare un servizio
4. Parlare con un consulente

Questa sezione deve essere molto visibile perché riduce la confusione.

## Priorità 3 — Tre macro-aree bilanciate

Le tre macro-aree devono avere lo stesso peso visivo:

- Veicoli in vendita
- Noleggio auto e moto
- Autolavaggio e officina

Regola: nessuna card deve essere più grande delle altre nella prima versione.

# Distribuzione visiva consigliata

## Sezione macro-aree

Desktop:

```
[ Vendita ] [ Noleggio ] [ Servizi ]
```

Mobile:

```
[ Vendita ]
[ Noleggio ]
[ Servizi ]
```

Ogni card deve avere:

- immagine o icona
- titolo chiaro
- testo breve
- CTA specifica

# Sezione veicoli in evidenza

Questa sezione non deve sembrare solo vendita o solo noleggio.

Consiglio: mostrare una selezione mista.

Esempio desktop:

```
[ Auto vendita ] [ Auto noleggio ] [ Moto vendita ] [ Moto noleggio ]
```

Se non ci sono abbastanza veicoli, usare filtri/tab:

```
Tutti | Vendita | Noleggio | Auto | Moto
```

# Sezione servizi

I servizi non devono essere messi troppo in basso, altrimenti sembrano secondari.

Posizione consigliata:

- dopo i veicoli in evidenza
- oppure prima dei veicoli se il cliente vuole spingere l’idea di piattaforma completa

Struttura:

```
[ Autolavaggio ] [ Diagnosi ] [ Manutenzione ] [ Officina ]
```

# CTA persistenti

## Desktop

- WhatsApp nel header
- Pulsante Contattaci
- Chat widget in basso a destra

## Mobile

- Sticky bottom bar con:
    - WhatsApp
    - Chiama
    - Chat

Questa soluzione aumenta le azioni rapide da mobile.

# Bilanciamento vendita / noleggio / servizi

Per non dare priorità sbagliata:

- Hero neutra
- CTA bilanciate
- Macro-card uguali
- Eventi analytics separati
- Nessuna categoria evidenziata come “principale” nella prima versione

Dopo 30-60 giorni, la gerarchia potrà essere adattata sulla base dei dati reali.

# Layout homepage consigliato

```
1. Header
2. Hero section
3. Quick need selector
4. Macro-aree: vendita / noleggio / servizi
5. Veicoli in evidenza
6. Servizi in evidenza
7. Perché scegliere Autopertutti
8. Come funziona
9. Supporto / Chat AI
10. CTA finale
11. Footer
```

# Note copywriting

Il tono deve essere:

- chiaro
- concreto
- affidabile
- non troppo aggressivo
- orientato all’aiuto

Evitare frasi troppo generiche come:

```
La migliore soluzione per la tua auto
```

Preferire frasi specifiche:

```
Trova un veicolo, richiedi un noleggio o prenota un servizio in pochi passaggi.
```

# Errori da evitare

- Mettere solo auto in hero
- Nascondere i servizi in fondo
- Usare troppe CTA diverse nello stesso blocco
- Form lunghi
- Prezzi non chiari
- Nessun contatto rapido da mobile
- Homepage troppo lunga senza scelta iniziale

# Raccomandazione finale

La prima versione deve essere neutra e misurabile. La homepage deve guidare, non decidere al posto dell’utente.

La priorità non è spingere un servizio, ma capire quale servizio l’utente cerca e portarlo rapidamente all’azione corretta.
